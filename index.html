                [media type]           [media features]

    Медиа-запросы

    Media type - тип устройства

    • all - все устройства (по умолчанию)
    • speech - скринридеры
    • print - принтеры и предпросмотр
    • screen - устройства с экраном

    Медиа-запросы

    Media features - технические характеристики устройства (mdn)

    • width, max-width - ширина вьюпорта
    • height, max-height - высота вьюпорта
    • orientation - ориентация (portrait | landscape)
    • resolution - разрешение
    • prefers-color-scheme - тема (темная или светлая)
    • ...

    Медиа-запросы

    Логический оператор

    • not - логическое «НЕ»
    • and - логическое «И»
    • , - логическое «ИЛИ»
    • only - сейчас можно не использовать

    Медиа-запросы

    @media screen and (min-width: 768px) -> @media screen
    @media only screen and (min-width: 768px) -> @media only

    Пиксель - это не пиксель

    Физический пиксель (physical pixel) - ячейка на матрице

    Виртуальный пиксель (device independent pixel) - независимый от устройства пиксель

    Пиксельное соотношение (pixel ratio) - отношение между физическим и виртуальным пикселем

    Пиксель - это не пиксель

    iphone 8

    Вьюпорт

    Viewport - видимая пользователю область страницы

    Вьюпорт

    <meta name="viewport" content="initial-scale=1, ...">

    • initial-scale - первоначальный масштаб веб-страницы
    • width - ширина вьюпорт
    • height - высота вьюпорт
    • user-scalable=no/yes - может ли пользователь масштабировать страницу
    • minimum-scale - минимальный масштаб вьюпорт
    • maximum-scale - максимальный масштаб вьюпорт

    По-умолчанию, css-ширина вьюпорта в мобильном браузере - 980px

    Вьюпорт

    <meta name="viewport" content="width=device-width, initial-scale=1">

    • width=device-width - вьюпорт равен ширине экрана в пикселях CSS
    • initial-scale=1 - масштаб при первой загрузке - 100%

    Что использовать - em, rem или px?

    Что использовать - em, rem или px?

    • em - если свойства масштабируются относительно font-size компонента;
    • rem - если свойства масштабируются относительно root font-size;
    • px - если нужна pixel-perfect верстка под каждый breakpoint, или если не знаете что лучше

    Когда загружать картинки?

    Картинки

    • Браузер парсит html и css построчно
    • Как только встречает картинку - добавляет в очередь загрузки
    • Когда приходит очередь, загружает картинку

    Картинки. Инлайн

    Важные (критичные) элементы интерфейса, которые надо сразу показать можно не грузить отдельно

                    <img src="data:image/svg+xml;base64,[data]">
      Минусы base64:
    • Большой размер, примерно на 30% больше, чем бинарный
    • Раздувает HTML

    Картинки. Img. Ленивая загрузка

    Ленивая загрузка - стратегия, при которой не критические ресурсы загружаются в тот момент, когда они нужны.

    Для ленивой загрузки у img есть аттрибут loading. Он доступен с Chrome 77

    Такая картинка загрузится, когда она окажется во вьюпорте пользователя.
